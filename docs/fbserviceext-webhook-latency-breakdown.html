FBServiceExt Webhook ACK Latency Breakdown

Current ACK path after the latest refactor: voting is stateless, ActiveShowId is runtime-controlled, and business-side payload validation or DB writes no longer belong to the synchronous webhook response.

Synchronous path that must finish before HTTP 200

1 Read request body + basic size guard
The controller reads UTF-8 body bytes and rejects oversized envelopes early.
2 Inspect payload shape
Classify the incoming body into message/postback/forgetme/garbage so cheap early gates can happen before publish.
3 Validate webhook signature
Meta webhook signature validation still belongs to the ACK path, because untrusted bodies must not be accepted into the queue.
4 Apply early API gates
  • garbage payload → acknowledge and drop
  • voting disabled → acknowledge and drop regular vote traffic
  • #forgetme still passes
5 Accept incoming connection + publish raw ingress envelope
The API writes the raw envelope to RabbitMQ and only measures publish-side latency here.
6 Return HTTP 200
Once the raw envelope is accepted and published, the webhook response is complete.

What is intentionally out of ACK path now

background only Voting business flow
  • GET_STARTED gate check against VotingStarted + ActiveShowId
  • cooldown lookup and cooldown write
  • signed VOTE1 carousel generation
  • signed CONFIRM1 parse and expiration validation
  • signed ERASE1 parse for #forgetme
  • AcceptedVotes DB insert
  • NormalizedEvents optional persistence
  • outbound Meta/local send
latency interpretation How to read current metrics
If ACK p95 is healthy but business completion is poor, the bottleneck is no longer API ingress. It is somewhere after raw publish: queue pressure, worker CPU, cooldown branch, confirmation branch, outbound send, or DB latency.

The practical consequence of the latest design is clear: webhook latency should be interpreted as API admission latency, not as full vote completion latency. Full completion belongs to the background workers and is shaped by ActiveShowId, signed payload validation, cooldown, DB persistence, and outbound delivery.
