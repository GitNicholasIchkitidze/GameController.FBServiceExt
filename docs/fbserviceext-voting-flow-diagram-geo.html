FBServiceExt ხმის მიცემის ბიზნეს ფლოუ

განახლებული სქემა session-free ხმის მიცემისთვის: signed payload-ები, ActiveShowId, show-scoped cooldown, stateless confirmation და არჩევითი NormalizedEvents persistence რეჟიმი.
შეტყობინება
ინიციატორი ან ჩვეულებრივი text command
payload
signed button payload: VOTE1 / CONFIRM1 / ERASE1
runtime gate
VotingStarted + ActiveShowId + cooldown
audit
NormalizedEventStorageMode: Full / Minimal / Disabled

1. ხმის მიცემის დაწყება

message GET_STARTED შემოდის
Controller ACK path მხოლოდ იღებს webhook-ს და queue-ში აგზავნის. რეალური voting logic უკვე worker-ში სრულდება.
runtime gate გადაამოწმე voting gate
  • თუ VotingStarted=false → voting inactive text
  • თუ ActiveShowId ცარიელია → voting inactive text
  • თუ cooldown key არსებობს → cooldown text
signed vote payload გაგზავნე candidate carousel
თითო ღილაკზე იწერება signed payload: VOTE1:{showId}:{candidateId}:{signature}
signature მიბმულია page/recipient კონტექსტზეც, რომ forged payload არ მიიღოს.
audit NormalizedEvents persistence
  • Full → ყველა normalized event ინახება
  • Minimal → მხოლოდ GET_STARTED და #forgetme
  • Disabled → საერთოდ არ იწერება

2. კანდიდატის არჩევა და დადასტურება

VOTE1 candidate payload შემოწმება
  • signature სწორი უნდა იყოს
  • showId უნდა ემთხვეოდეს მიმდინარე ActiveShowId-ს
  • თუ payload invalid/staleა → cooldown check, მერე ან cooldown text ან ახალი carousel
feature flag confirmation ყველასთვის ან პირდაპირი ჩაწერა
default-ზე confirmation ყველასთვის საჭიროა. თუ flag გამორთულია, flow პირდაპირ გადადის cooldown check → vote accept → success text branch-ში.

CONFIRM1 დადასტურების სამი ღილაკი
ერთი სწორი და ორი decoy. payload ფორმატი: CONFIRM1:{showId}:{candidateId}:{issuedAt}:{action}:{signature}
  • issuedAt ≤ 120 წამი
  • action=YES → vote write branch
  • არასწორი action → ახალი carousel
  • expired → timeout text
  • invalid/malformed/signature fail → cooldown check, მერე recovery
final write AcceptedVotes-ში ჩაწერა
  • cooldown key იწერება showId + recipientId + userId-ზე
  • AcceptedVotes ინახავს ShowId-ს და UserAccountName-ს
  • თუ FB account name ვერ მოიძებნა → UserId + " NOT FETCHED FROM FB"

3. #forgetme და side state

message #forgetme request
ახლა ესეც stateless flow-ია. session store აღარ გამოიყენება.
ERASE1 signed erase confirmation
ფორმატი: ERASE1:{userId}:{issuedAt}:{action}:{signature}
ShowId აქ არ გამოიყენება, რადგან deletion user-ს ეკუთვნის და არა კონკრეტულ შოუს.
erasure სრული წაშლა logs-ის გარდა
  • AcceptedVotes
  • NormalizedEvents
  • Redis cooldown keys
  • account-name cache
  • legacy keys იმავე user scope-ზე
ოპერატორის კონტროლი ActiveShowId runtime state
VotingStarted მარტო აღარ კმარა. ახალი show იწყება მხოლოდ მაშინ, როცა operator board-ზე ახალი ActiveShowId ჩაიწერება.
ძირითადი ცვლილება: ძველი OptionsSent / ConfirmationPending / CooldownActive session snapshot-ები voting flow-დან მთლიანად ამოღებულია. ახლა მოქმედებს stateless signed payload-ები, show-scoped cooldown და runtime gate: VotingStarted + ActiveShowId.
